# Strumly - Додаток для спільного виконання музики (Jam Sessions)

Цей проєкт складається з двох частин:
1. **Бекенд (NestJS)**: знаходиться в каталозі `strumly-backend`.
2. **Клієнт (Flutter)**: знаходиться в каталозі `strumly`.

---

## 🚀 Як запустити проєкт

### Крок 1: Запуск Бекенду (NestJS)

1. Перейдіть до директорії бекенду:
   ```bash
   cd ../strumly-backend
   ```
2. Встановіть залежності (якщо ще не встановлено):
   ```bash
   npm install
   # або для чистого встановлення:
   npm ci
   ```
3. Створіть та налаштуйте файл `.env` у корені `strumly-backend` (перевірте підключення до бази даних PostgreSQL).
4. Запустіть сервер у режимі розробки:
   ```bash
   npm run start:dev
   ```
   Сервер запуститься за адресою `http://localhost:3000`.

---

### Крок 2: Запуск Android-емулятора

1. Перегляньте список доступних емуляторів:
   ```bash
   flutter emulators
   ```
2. Запустіть обраний емулятор (наприклад, `StrumlyPhone` або `Medium_Phone_API_36.1`):
   ```bash
   flutter emulators --launch StrumlyPhone
   ```

---

### Крок 3: Запуск Мобільного Додатку (Flutter)

1. Перейдіть до директорії додатку:
   ```bash
   cd strumly
   ```
2. Перевірте список підключених пристроїв (переконайтеся, що емулятор запущено та підключено):
   ```bash
   flutter devices
   ```
3. Запустіть додаток на потрібному пристрої (наприклад, на емуляторі `emulator-5554` або macOS/Chrome):
   ```bash
   flutter run -d emulator-5554
   ```

---

## 🛠️ Функціонал Jam Session та Метронома

- **Запрошення друзів у Jam**: В екрані детальної інформації про пісню або плейліст натисніть кнопку **"Запросити"**. Буде автоматично підключено Socket-з'єднання чату та надіслано повідомлення `[JAM_INVITE]код_сесії`. Після цього хост перейде на екран джему.
- **Кнопка "Join"**: Отримувач запрошення в чаті побачить повідомлення у вигляді картки з кнопкою **"Приєднатись"**. Натискання на неї підключає користувача до джему.
- **Метроном для всіх пісень**: Навіть якщо для обраної пісні в базі даних не встановлено BPM (значення `null`), для хоста автоматично встановиться дефолтне значення **120 BPM**. Хост може коригувати BPM кнопками `+` та `-` під час сесії.
- **Синхронізований автоскролінг**: При старті автоскролінгу хостом відтворюється 4-кратний відлік метроному (count-in), після чого починається синхронний скролінг тексту для всіх учасників.
